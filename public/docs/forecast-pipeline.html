Two distinct flows. Create drafts and persists a forecast; Analyze is a separate, repeatable step that fetches fresh context and produces the calibrated Oracle estimate. Hover any node for the exact function & file.

⟶ Create ends at an ACTIVE forecast  ·  Analyze runs later, on demand, as many times as you like

🟥 Moderation runs twice

Once when the express draft is generated, and again on persist. The manual path skips the draft, so its only moderation is on POST /api/forecasts.

🟪 The "math" is in Analyze

The draft's probability is just an LLM suggestion. The calibrated, credibility-weighted estimate (mean/std/CI) comes from the external Oracle's /forecast — only in the Analyze flow.

▦ Translate & embed are background

They're fire-and-forget jobs fired after the row is saved — the response returns before they finish. They are not an inline stage between censorship and search.
