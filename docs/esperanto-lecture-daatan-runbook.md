# Esperanto lecture: DAATAN interactive segment (runbook)

Operational runbook for a hybrid talk (~20–30 in room, ~5 on Zoom). Tone **B**: movement health — measurable, checkable claims. This document implements the lecture plan (audience, structure, logistics, drafts, slides, rehearsal); it does **not** replace creating forecasts in the product.

---

## Audience and format

- **In room:** ~20–30 Israeli Esperantists.
- **Remote:** ~5 on Zoom.
- **Tone B:** Story-first delivery, but every claim must be **resolvable** from named sources (official sites, Wikimedia stats, UEA releases, predefined media lists).
- **Scale:** ~25–35 total — assign someone to **monitor Zoom chat** during live co-creation so remote wording and votes are not lost.

**Goals:** Co-create **one** forecast live (wording + deadline + resolvers), or use a tight variant; keep a **second forecast** pre-seeded so Zoom participants can commit without waiting on latency.

**Your forecasts (confirmed):**

1. **Wikipedia** — already created in DAATAN (Esperanto Wikipedia threshold or similar).
2. **Participants next year** — create in DAATAN before or during the segment using the template below.

---

## What makes a good lecture forecast

- **One sentence, falsifiable:** yes/no or a clear numeric threshold by a **calendar date**.
- **Resolution source named up front:** e.g. Special:Statistics URL, named UEA page, named event page.
- **Resolvers named:** e.g. you + one volunteer (simplest for a live event).

## Chosen pair: themes and resolution rules

### Forecast 1 — Esperanto Wikipedia (you created this)

**Intent:** Public article count on `eo.wikipedia.org` by a deadline.

**Resolution (standard):** Use [Esperanto Wikipedia Special:Statistics](https://eo.wikipedia.org/wiki/Special:Statistics). Resolve **Yes** if the **Articles** (content pages) count is **≥ N** by **23:59 UTC** on the stated date; otherwise **No**. Describe **N** and **DATE** in the forecast body so nothing is ambiguous.

**Source:** `https://eo.wikipedia.org/wiki/Special:Statistics` (or cite a Wikimedia stats page if you specify it consistently in the forecast).

### Forecast 2 — Number of participants “next year” (template for your new forecast)

Use this only if you can point to **one official place** where the number will appear (event site, UEA, registration PDF on a named domain).

**Question (template):** The **official published attendance or registration count** for **[EVENT NAME, YEAR]** will be **≥ X participants** (or **≥ X unique registrants**), as stated on **[exact page type: e.g. uea.org page, event homepage, official PDF URL pattern]** by **23:59 UTC on [RESOLUTION CHECK DATE]**.

**Resolution:** **Yes** if a number **≥ X** appears on the cited source by the check date; **No** if only a lower number is published or no authoritative number appears. If the metric is ambiguous (e.g. “visitors” vs “delegates”), pick **one** metric in the forecast text and stick to it.

**Risk:** If organizers might not publish a number, prefer a different institutional signal or narrow **X** and the event so publication is likely.

## Live segment structure (~12–18 minutes)

1. **30–60s hook:** Personal “I told you so” / quantum bet story → “today we put something in writing that time can judge.”
2. **Show DAATAN:** Projector: **Forecasts** feed; optionally one **resolved** forecast if you have a good example (keep calibration explanation to one sentence).
3. **Optional co-creation:** If time allows — “What is one **measurable** thing we care about for Esperanto in the next 12–24 months?” Crowdsource **up to three** wordings (room + Zoom chat); vote; publish **or** skip if you already have two strong forecasts published.
4. **Second forecast:** Display the **participants** forecast (pre-created). Ask room + Zoom to **open the link**, **sign in**, **commit** if they have a view.
5. **Close:** “You’re on the witness list.” QR to `https://daatan.com` and, if shown, direct links to both forecasts.

## Hybrid logistics

- **Zoom:** Share a **browser tab** (not whole desktop). Spotlight speaker when needed.
- **Room:** QR on slide to `https://daatan.com` and/or **permalink** to each forecast (fill in the night before).
- **Accessibility:** Read the **final forecast text aloud** once for Zoom; paste the same text into Zoom chat.
- **Roles:** You = facilitator + resolver; optional **second resolver** (volunteer) for clarity.

## Risks to avoid

- Vague verbs (“more popular”) — use counts, dates, named sources.
- Political debate inside resolution — stick to **documented facts** the forecast text already defines.
- Long co-creation — cap wording debate at **4–5 minutes**; you trim to one crisp sentence.
